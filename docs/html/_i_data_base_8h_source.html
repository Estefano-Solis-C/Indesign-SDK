653  //------------------------------------------------------------------------------------
654  // Creates a new persistent object in the database. The caller specifies the desired
655  // UID of the new object. This will succeed under the following conditions:
656  // 1. The requested UID is in the valid range - (currently: 1 <= UID <= 2^31-1)
657  // 2. The UID has either never been used, or can be re-used
658  // A UID that was used before can only be re-used provided that it does not exist
659  // in either the temporary file or the user file. This requires at least calling
660  // DeleteUID and DestoryUID, and performing a save operation afterwards if the
661  // database protection level is greater than kNoProtection.
662  //
663  // There may be three outcomes when calling this function. Callers should be prepared:
664  //
665  // If the function succeeds, the object is created and the function returns kTrue
666  //
667  // If the function fails because the UID isn't available to be used (outside of the
668  // valid range, not available for re-use yet), the function returns kFalse but the
669  // database error state is not set
670  //
671  // If the function fails for some more insidious reason, it returns kFalse and the
672  // database error state is set appropriately
674  virtual bool16 PlacementNewUID(ClassID clsID, UID id) = 0;
